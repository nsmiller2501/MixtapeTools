# /grill-with-docs

Like `/grill-me`, but domain-aware. Before the interview begins, Claude reads the project's `agent_memory/CONTEXT.md` glossary and any existing Architecture Decision Records (ADRs), then challenges your plan against that documented language. When a term gets resolved or a hard architectural decision gets made during the session, Claude writes it into `CONTEXT.md` or a new ADR immediately — so the documentation stays current as decisions crystallize rather than being batched at the end.
